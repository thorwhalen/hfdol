hfdol.deploy

Deploy webapps to HuggingFace Spaces.

This module wraps huggingface_hub with opinionated, idempotent helpers for publishing a Python package’s webapp to a Docker-SDK Space. The primary entry point is deploy_webapp(), which composes the smaller helpers into a full create-or-update + restart cycle.

Functions are designed to be composable: each does one thing, fails loudly, and prints what it’s about to do. The module assumes you already have a package on PyPI (or a Dockerfile that knows how to install it).

Quick reference:

>>> from hfdol.deploy import deploy_webapp  
>>> deploy_webapp(                          
...     repo_id="thorwhalen/typola",
...     source_dir="/path/to/staging",
... )

For ad-hoc operations:

>>> from hfdol.deploy import factory_reboot, wait_for_build  
>>> factory_reboot("thorwhalen/typola")     
>>> wait_for_build("thorwhalen/typola")

Token resolution: Operations need a write-scoped HF token. By default, ensure_write_token() reads HF_WRITE_TOKEN from the environment, then falls back to sourcing ~/.keys (a shell file that exports it). If you keep your write token elsewhere, pass token=... to any function explicitly.

hfdol.deploy.create_or_update_space(repo_id: str, *, sdk: str = 'docker', private: bool = False, token: str | None = None, exist_ok: bool = True)[source]

Create an HF Space if it doesn’t exist; otherwise return its info.

Idempotent. Safe to call before every deploy.

Parameters:

  • repo_id – owner/space-name.

  • sdk – Space SDK — docker (recommended for custom apps), gradio, streamlit, static.

  • private – If True, create as private (only matters on first creation).

  • token – Write token; defaults to ensure_write_token().

  • exist_ok – If True (default), an already-existing repo is fine.

Returns:

SpaceInfo for the (now-existing) Space.

hfdol.deploy.deploy_webapp(repo_id: str, source_dir: str | Path, *, sdk: str = 'docker', private: bool = False, token: str | None = None, message: str = 'Deploy webapp', ignore_patterns: Iterable[str] = ('node_modules/**', '__pycache__/**', '*.pyc', '.DS_Store', '.git/**'), delete_patterns: Iterable[str] | None = None, rebuild: bool = True, wait: bool = True, timeout: int = 900) → BuildResult | None[source]

End-to-end: create-if-missing → upload → factory reboot → wait for build.

The most common deploy entry point. source_dir should contain everything that goes on the Space repo: Dockerfile, README.md (with HF YAML frontmatter), and your app source (e.g., webapp/api/, webapp/ui/dist/).

Parameters:

  • repo_id – owner/space-name.

  • source_dir – Local staging directory.

  • sdk – Space SDK (only matters on first creation).

  • private – Create as private (only matters on first creation).

  • token – Write token; defaults to ensure_write_token().

  • message – Commit message.

  • ignore_patterns – Glob patterns to skip during upload.

  • delete_patterns – Glob patterns to delete from the Space before uploading (e.g., to clear stale build artifacts).

  • rebuild – If True (default), trigger a factory reboot after upload. Set False to skip — useful when only README/metadata changed.

  • wait – If True (default), poll until the build settles.

  • timeout – Max seconds to wait for the build.

Returns:

BuildResult if wait=True, else None.

hfdol.deploy.ensure_write_token(*, env_var: str = 'HF_WRITE_TOKEN', keys_file: Path = PosixPath('/home/runner/.keys')) → str[source]

Return a write-scoped HF token; source ~/.keys if needed.

HF Spaces operations (create, upload, restart) need a write token. The default HF_TOKEN is often read-only. Convention: keep the write token in ~/.keys as export HF_WRITE_TOKEN=hf_....

Parameters:

  • env_var – Environment variable name to read first.

  • keys_file – Shell file to source as fallback (must export env_var).

Returns:

The token string.

Raises:

SystemExit – If no token can be resolved.

hfdol.deploy.factory_reboot(repo_id: str, *, token: str | None = None) → None[source]

Trigger a from-scratch rebuild of the Space (busts Docker layer cache).

A normal restart reuses cached Docker layers. A factory reboot blows the cache away — required when pip install should re-fetch a new package version, or when system deps in the Dockerfile changed.

Parameters:

  • repo_id – owner/space-name.

  • token – Write token; defaults to ensure_write_token().

hfdol.deploy.render_pypi_webapp_dockerfile(package: str, *, extras: str = 'web', version_spec: str = '', python_version: str = '3.12', api_module: str = 'webapp.api.main:app', port: int = 7860, api_dir: str = 'webapp/api', ui_dist_dir: str = 'webapp/ui/dist', extra_run_lines: Iterable[str] = ()) → str[source]

Render a Dockerfile for the canonical “PyPI package + FastAPI + React” pattern.

Assumes: - Your Python package is on PyPI (with optional [extras]). - Your FastAPI app is in api_dir/ (will be COPYed into the image). - Your React UI is pre-built in ui_dist_dir/ (will be COPYed in).

Why pre-build the UI locally instead of in the Dockerfile? Because package-lock.json may have file: deps to a local monorepo that doesn’t exist on the build machine. Pre-building sidesteps the issue.

Parameters:

  • package – PyPI package name.

  • extras – Extras spec (without brackets), e.g. "web" → package[web]. Pass empty string to omit.

  • version_spec – PEP 440 version specifier, e.g. "~=0.1.1". Empty string means “latest”.

  • python_version – Python base image tag ("3.12" → python:3.12-slim).

  • api_module – module.path:app_attr for uvicorn.

  • port – Port to expose (HF Spaces uses 7860 for Docker SDK).

  • api_dir – Local path of the FastAPI source, relative to the Space repo root.

  • ui_dist_dir – Local path of the pre-built React dist, relative to the Space repo root.

  • extra_run_lines – Additional RUN lines to insert (e.g., to pre-cache datasets so cold start is sub-second).

Returns:

The Dockerfile contents as a string.

hfdol.deploy.render_space_readme(title: str, *, color_from: str = 'blue', color_to: str = 'green', sdk: str = 'docker', port: int = 7860, pinned: bool = False, body: str = '') → str[source]

Render an HF Space README with the YAML frontmatter HF requires.

The frontmatter sets the Space’s metadata (sdk, port, etc.) — required even for Docker SDK Spaces. Body is appended after.

Parameters:

  • title – Display title for the Space.

  • color_from – Gradient start color (HF picks from a small palette).

  • color_to – Gradient end color.

  • sdk – "docker", "gradio", "streamlit", "static".

  • port – app_port (Docker SDK uses this; ignored otherwise).

  • pinned – Whether to pin to the user’s HF profile.

  • body – Markdown content to append after the frontmatter.

hfdol.deploy.stage_webapp(*, staging_dir: str | Path, api_src: str | Path, ui_dist_src: str | Path, api_dir_in_space: str = 'webapp/api', ui_dist_dir_in_space: str = 'webapp/ui/dist', dockerfile_text: str | None = None, readme_text: str | None = None, extra_files: dict[str, str] | None = None) → Path[source]

Lay out the contents of a Space repo in a local staging directory.

Idempotent: re-running replaces the API and UI dirs but reuses what’s already there for things you didn’t pass (e.g., an existing Dockerfile).

Parameters:

  • staging_dir – Where to assemble the Space contents.

  • api_src – Local source directory for the FastAPI app.

  • ui_dist_src – Local source directory for the pre-built React dist.

  • api_dir_in_space – Path inside the Space repo for the API.

  • ui_dist_dir_in_space – Path inside the Space repo for the UI dist.

  • dockerfile_text – Dockerfile contents (use render_pypi_webapp_dockerfile()). If None, only writes one if the staging dir doesn’t already have it.

  • readme_text – README contents (use render_space_readme()). Same conditional behavior as dockerfile_text.

  • extra_files – {relative_path: contents} extras (e.g., .gitignore).

Returns:

The staging directory path.

hfdol.deploy.upload_app_dir(repo_id: str, folder_path: str | Path, *, message: str = 'Update app contents', token: str | None = None, ignore_patterns: Iterable[str] = ('node_modules/**', '__pycache__/**', '*.pyc', '.DS_Store', '.git/**'), delete_patterns: Iterable[str] | None = None) → None[source]

Upload a local folder to an HF Space, replacing the live files.

Wraps huggingface_hub.HfApi.upload_folder() with sensible ignore patterns and a printed plan. Existing files on the Space that are not in the local folder are NOT deleted unless you pass delete_patterns.

Parameters:

  • repo_id – owner/space-name.

  • folder_path – Local directory to upload.

  • message – Commit message on the Space repo.

  • token – Write token; defaults to ensure_write_token().

  • ignore_patterns – Glob patterns to skip.

  • delete_patterns – Optional glob patterns of remote files to delete before upload (e.g., ["webapp/ui/dist/**"] to ensure stale build artifacts are removed).

hfdol.deploy.wait_for_build(repo_id: str, *, token: str | None = None, timeout: int = 900, poll_interval: int = 15, on_stage_change: Callable[[str, int], None] | None = None) → BuildResult[source]

Poll Space status until it lands in a terminal state or times out.

Terminal states: RUNNING, RUNNING_BUILDING (success); BUILD_ERROR, RUNTIME_ERROR (failure). Times out after timeout seconds.

Parameters:

  • repo_id – owner/space-name.

  • token – Write token (read access also fine); defaults to env.

  • timeout – Max seconds to wait.

  • poll_interval – Seconds between polls.

  • on_stage_change – Callback (stage, elapsed_seconds) -> None invoked each time the stage transitions. Default prints to stdout.

Returns:

BuildResult.
